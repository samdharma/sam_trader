4. Deploy & Container Lifecycle

# Start all 6 containers
./deploy.sh start

# Stop all
./deploy.sh stop

# Rebuild images + start
./deploy.sh --build start

# Restart a single container (e.g., after config change)
docker compose -f docker/docker-compose.yml restart sam-trader

# View container status
docker compose -f docker/docker-compose.yml ps

Tip: add alias sam='docker exec sam-services sam' to your shell profile.

5. Status & Health

sam status
Docker ps filtered to sam-* containers (name, state, health, ports).
sam health
Deep health check: PostgreSQL, Redis, Futu OpenD connectivity, Nautilus engine heartbeat.
sam data-health
Verify market-data bar flow end-to-end after restart or market switch. Checks latest bar timestamps per instrument against expected freshness.
sam deploy-window
Check whether current time is inside the safe deploy window (default 05:00–08:00 HKT).
sam logs [service]
Tail logs for a specific service or all sam-* containers.
sam version
Show deployed version: git tag/commit + build timestamp.

Automatic (preferred)

The MarketSchedulerActor runs inside sam-trader and publishes switch requests automatically:

  • HK → US: 16:00 HKT (HK market close)
  • US → HK: 04:00 HKT (US close + maintenance window open)

No manual intervention needed — the scheduler waits for the maintenance window, runs gate checks, and triggers the switch.

Manual (emergency / test)

sam switch-market US
sam switch-market HK
sam switch-market US --timeout 300
Request market switch and wait for orchestrator to complete the full flow (save → env update → container restart → state_loaded). Default timeout 120s.
Switch only inside the maintenance window (04:00–07:00 HKT). The RestartOrchestrator enforces this gate — requests outside the window are rejected.

7. Pre-Market & Pipeline

sam pipeline [--market US|HK]
Full pre-market pipeline: gap scan → AI scoring → bundle generation → readiness report. If --market is omitted, uses the current MARKET from .env.
sam gapscan --market US --pass 1
sam gapscan --market HK --pass 1
sam gapscan --market US --pass 2
Dual-broker pre-market gap scanner. Pass 1: initial scan from premarket_watchlist.yaml. Pass 2: re-scan narrowed candidates. Results cross-validated between primary and secondary brokers (US: Futu + IB; HK: Futu only).
sam watchlist [--market US|HK]
Show the current pre-market watchlist universe for the given market.
sam readiness-report
Generate the full SOD readiness report (7 checks: broker connectivity, quote flow, instruments resolved, account status, bundles loaded, Redis/PG health, calendar trading day). Results written to Redis sam:readiness:{market}:{date}.
sam readiness --market HK
Read the published readiness report from Redis (previously generated by ReadinessCheckerActor or readiness-report).

8. Bundle Management

Bundles define active strategies. Filtered by market field at boot so only strategies matching the current MARKET are loaded.

CLI

sam validate-bundles
Validate bundle YAML: schema check + strategy class import + optional backtest gate.
sam bundle-diff
Show pending bundle changes compared to the last deployed snapshot in Redis.

Runtime Bundle Control (via Redis pub/sub)

The BundleController inside sam-trader subscribes to Redis channels for hot-reload of strategies without restart:

# Load a new bundle at runtime
redis-cli -h sam-redis PUBLISH sam:bundle:load '{"id":"my-strat","venue":"FUTU","market":"HK","strategy":{"path":"sam_trader.strategies.orb:OrbStrategy","config":{"instrument_id":"00700.HKEX","bar_type":"00700.HKEX-15-MINUTE-LAST-EXTERNAL","first_candle_minutes":15,"trade_size":100}},"bracket":{"stop_loss_ticks":20,"take_profit_ticks":60},"risk":{"max_position":1000,"max_daily_loss":5000}}'

# Unload a strategy by ID
redis-cli -h sam-redis PUBLISH sam:bundle:unload '{"strategy_id":"OrbStrategy-0"}'
Bundle fields: id, enabled, venue (FUTU|IB), market (US|HK), strategy.path, strategy.config, bracket.stop_loss_ticks, bracket.take_profit_ticks, risk.max_position, risk.max_daily_loss. See config/bundles.example.yaml for full annotated reference.

10. Emergency & Safety

Kill / Halt / Resume

sam kill
EMERGENCY Cancel ALL orders + halt trading immediately. Use when you need to flatten everything NOW.
sam halt
HALT Halt trading: cancel open orders, switch to position-close-only mode. Strategies remain loaded but cannot submit new orders.
sam resume
RESUME Resume trading: clear halt state, strategies can submit orders again.

Cache Flush

sam flush-cache --force
USE SPARINGLY Emergency Redis cache flush. Removes ALL keys from the Redis DB — clears stale orders, strategy state, and cached data. Use when the node cannot start because of orphaned orders persisted across restarts.

Safety Env Vars

SAFETY_MAX_DAILY_LOSS=0Max daily P&L loss (0 = disabled)
SAFETY_MAX_REJECTION_STREAK=3Order rejections before auto-halt
SAFETY_CONNECTIVITY_TIMEOUT_SECS=60Stale connection detection
RISK_BYPASS=0Set to 1 to bypass all risk checks (paper only!)

13. Troubleshooting

SymptomLikely CauseFix
sam readiness shows broker FAIL Futu OpenD or IB Gateway not healthy docker compose restart sam-futu-opend sam-ib-gateway
Zero bundles loaded at boot MARKET in .env mismatches bundle market field Check MARKET= in .env vs market: field in config/bundles.yaml
Bars stale after market switch State loaded but execution client not reconnected sam flush-cache --force then restart
Scanner discrepancies >1% between brokers Futu vs IB price drift (pre/post-market spread differences) Run sam data-health; disable secondary broker in config/gap_scanner.yaml if persistent
Market switch request rejected Outside maintenance window (04:00–07:00 HKT) Wait for window or use sam deploy-window to check timing
Strategies not trading after resume Halt state not fully cleared or bundle not re-loaded Run sam resume then re-publish bundle via Redis if needed
Redis connection refused from CLI sam-redis container not running or network issue sam status → verify sam-redis is healthy; restart if needed
